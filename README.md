## 🏗️ Architecture

<div align="center">

### 🔄 Data Flow

1. **Frontend Applications** communicate with the backend via RESTful APIs
2. **Authentication Layer** handles JWT token validation and role-based access
3. **Business Logic** processes requests and applies validation rules
4. **Database Layer** stores and retrieves data using MongoDB
5. **External Services** handle payments, emails, file storage, and PDF generation

## 🚀 Quick Start

<div align="center">

### ⚡ Get up and running in 5 minutes!

</div>

```bash
# 1. Clone the repository
git clone https://github.com/AAYUSH412/society-ease.git
cd society-ease

# 2. Install all dependencies
npm run install:all

# 3. Set up environment variables
cp backend/.env.example backend/.env.local
cp frontend/.env.example frontend/.env.local
cp admin/.env.example admin/.env.local

# 4. Start all services
npm run dev
```

<div align="center">

🎉 **That's it!** Your applications will be running at:
- **Backend API**: http://localhost:4000
- **Resident Portal**: http://localhost:3000  
- **Admin Dashboard**: http://localhost:3001

</div>

### 🔧 Step-by-Step Installation

<details>
<summary><b>📥 1. Clone & Setup</b></summary>

```bash
# Clone the repository
git clone https://github.com/AAYUSH412/society-ease.git
cd society-ease

# Install workspace dependencies
npm install

# Install dependencies for all applications
npm run install:all
```

</details>

<details>
<summary><b>⚙️ 2. Environment Configuration</b></summary>

```bash
# Copy environment files
cp backend/.env.example backend/.env.local
cp frontend/.env.example frontend/.env.local
cp admin/.env.example admin/.env.local

# Edit the environment files with your configurations
# See Configuration section below for details
```

</details>

<details>
<summary><b>🗄️ 3. Database Setup</b></summary>

```bash
# Start MongoDB service (if installed locally)
brew services start mongodb/brew/mongodb-community

# Or use MongoDB Atlas (cloud)
# Update MONGODB_URI in backend/.env.local
```

</details>

<details>
<summary><b>🚀 4. Start Development Servers</b></summary>

```bash
# Start all applications concurrently
npm run dev

# Or start individually
npm run dev:backend    # API Server (Port 4000)
npm run dev:frontend   # Resident Portal (Port 3000)
npm run dev:admin      # Admin Dashboard (Port 3001)
```

</details>

## ⚙️ Configuration

<div align="center">

### 🔐 Environment Variables

</div>

<details>
<summary><b>🖥️ Backend Configuration (.env.local)</b></summary>

```bash
# Server Configuration
PORT=4000
NODE_ENV=development

# Database
MONGODB_URI=mongodb://localhost:27017/society-ease

# JWT Secrets (Generate strong secrets)
JWT_SECRET=your-super-secret-jwt-key-min-32-chars
JWT_REFRESH_SECRET=your-refresh-token-secret-min-32-chars

# Email Service (Brevo/Sendinblue)
BREVO_API_KEY=your-brevo-api-key
BREVO_SMTP_HOST=smtp-relay.brevo.com
BREVO_SMTP_PORT=587
BREVO_SMTP_USER=your-brevo-smtp-login
BREVO_SMTP_PASS=your-brevo-smtp-password
FROM_EMAIL=noreply@yourdomain.com
FROM_NAME=Society Ease

# Payment Gateway (Razorpay)
RAZORPAY_KEY_ID=rzp_test_your_key_id
RAZORPAY_KEY_SECRET=your_razorpay_secret

# Image Storage (ImageKit)
IMAGEKIT_PUBLIC_KEY=public_your_imagekit_key
IMAGEKIT_PRIVATE_KEY=private_your_imagekit_key
IMAGEKIT_URL_ENDPOINT=https://ik.imagekit.io/your_endpoint

# Security
CORS_ORIGIN=http://localhost:3000,http://localhost:3001
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
```

</details>

<details>
<summary><b>🌐 Frontend Configuration (.env.local)</b></summary>

```bash
# API Configuration
NEXT_PUBLIC_API_URL=http://localhost:4000/api

# App Configuration
NEXT_PUBLIC_APP_NAME=Society Ease
NEXT_PUBLIC_APP_VERSION=1.0.0

# Payment Gateway
NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_test_your_key_id

# Feature Flags
NEXT_PUBLIC_ENABLE_REGISTRATION=true
NEXT_PUBLIC_ENABLE_GUEST_MODE=false

# Analytics (Optional)
NEXT_PUBLIC_GA_MEASUREMENT_ID=G-XXXXXXXXXX
```

</details>

<details>
<summary><b>🔧 Admin Configuration (.env.local)</b></summary>
